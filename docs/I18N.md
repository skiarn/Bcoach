# I18N Standard

This project uses a central UI translation system.

## Source Of Truth
- UI messages live in src/i18n/messages.ts
- Runtime locale/context lives in src/i18n/I18nProvider.tsx
- Text render helper widget lives in src/components/Trans.tsx

## Naming Convention For Keys
Use dotted keys with this shape:
- area.section.intent

Examples:
- nav.home
- home.importMany.title
- analyze.error.saveVideo
- history.groups.feedbackCount

Rules:
- Keep keys stable. Do not rename without migration.
- Use explicit intent names (title, subtitle, error, cta, hint, placeholder).
- Put shared keys under common.*.
- Keep all variable placeholders in braces, for example {count}, {videoName}.

## How To Add New Text
1. Add both sv and en entries in src/i18n/messages.ts.
2. In components/pages, use:
   - const { t } = useI18n()
   - t('your.key')
3. For simple inline render, use:
   - <Trans k="your.key" />
4. For dynamic values, use:
   - t('your.key', { count: 3 })

## What Not To Do
- Do not hardcode user-facing UI strings in JSX.
- Do not display raw internal/technical errors directly to users.
- Do not add locale-specific branching in components when a key can handle it.

## Error Message Policy
- Map surfaced errors to translation keys.
- Keep technical details in console logs.
- Show users localized, action-oriented messages.

## CI Check
Run:
- npm run check:i18n

This script flags new hardcoded UI literals in TSX files so untranslated text does not regress.
